HEALTHCARE CLAIMS FRAUD DETECTION

ABSTRACT:

Medical Insurance Fraud is very common these days. In this type of fraud‚ false or misleading information is provided to a health insurance company in an attempt to have them pay unauthorized benefits to the policy holder‚ another party‚ or the entity providing services. The offense can be committed by the insured individual or the provider of health services. Our objective is to interpret, implement and demonstrate designs and systems that leverages technologies like AI and ML to make an efficient healthcare system and maintain maximum accuracy on data of systems. We will use a combination of web technologies for better user interface and several Machine Learning algorithms to develop a system which can detect anomalies, help recover overpayment by user, can flag claims for potential Appeals and Grievances and help with provider payment preferences.

DESCRIPTION:

This project will reduce the organizational gaps between patients and healthcare system and the project will also be user friendly so that anyone can use it easily. In this application we will provide four main problem outlines as options to the user which are described as :-

  • Intake and pre-processing anomaly detection.(Predict anomaly)
      Detect anomalies and discrepancies in the healthcare claims intake / acquisition and preprocessing / data loading steps. Example- Alerts/ notifications can be sent out in case of table loading errors or where re-submissions are required.
  • Overpayment Recovery and adjustment analysis.(Overpayment Detection),
      Built an adjustments or corrections system to be used whenever Overpayment occurs from the side of the payer in the previous claim payment cycle for a physician/hospital.
  • Decision Flagging for potential appeals and grievances inflow.(Predict Fraud)
      Based on claim decisions made, we have built an AI/ML-driven analytics model that flags claims for the potential Appeals & Grievances (A&G) inflow and processing.
  • Categorization based on provider payment preferences.(Predict mode of payment)
      Based on provider payment preferences, we have built an AI/ML-driven analytics model that groups claims the Checks and Electronic Fund Transfer (EFT). Based on provider preferences, analytics and decision , support could further be applied on rolling out tailored suggestions and services to providers – to improve provider engagement, experience, and retention.
    • We have made a web application which will demonstrate the idea and take input from the user about the beneficiary which will include gender, state, Bill amount, reimbursement amount, Impatient claim number, outpatient claim number, inpatient claim amount, outpatient Claim amount. By clicking on button provided you will get your result.
    Technology Stack : Framework used – Google colab, HTML, CSS, Bootstrap, Github.
